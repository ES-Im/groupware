## _dept_
- 부서 코드와 부서 활성 여부 등 부서 기본 정보를 저장하는 엔티티

**속성**

| 필드명             | 비고           | 설명             | 
|-----------------|--------------|----------------|
| `id         `   | INT          |                |    
| `dept_code  `   | varchar(3)   | 부서코드 ex- '001' |    
| `dept_name  `   | varchar(100) | 부서명            |    
| `is_active  `   | bool         | 활성화 여부         |    
| `created_at `   | DATETIME     |                |    
| `updated_at `   | DATETIME     |                |    

**행위**
- `static registerDept()`: 부서 생성: deptCode, deptName
- `activate()`: 부서 활성화
- `deactivate()`: 부서 비활성화
- `renameDept()`: 부서 이름 변경

**규칙**
- 부서 비활성화는 해당 부서에 'ACTIVE' 상태인 사원이 없을 때만 가능하다.
- 신규 부서는 기본적으로 활성 상태이다.
- 비활성화 부서는 일반 조회 대상에서 제외된다.
- 부서코드는 부서 생성 후 변경하지 않는다.
- 부서 정보 수정 / 등록은 SystemRoleCode = `ADMIN`만 할 수 있다.

## _emp_
- 사원 계정 및 인적사항, 입사일 등 사원 기본 정보를 저장하는 엔티티
- Aggregate root

**회원 등록/수정 규칙**
- 신규 사원은 회원가입 시
    - [사번, 이름, 아이디, 패스워드]를 입력한다.
- 신규 생성 이후 SystemRoleCode에 따라 아래 정보를 수정하거나 등록한다.
    1) [가입승인] 
        - 권한 : SystemRoleCode = `ADMIN`
        - 가입 승인 시, EMP의 2가지 속성 변경
          - `Emp.status` - `ACTIVE`(활성화)로 설정
          - `Emp.SystemRoleCode` - `EMPLOYEE`로 설정
    2) [직원 정보 변경] 
        - 권한 : `Emp.SystemRoleCode` = `DEPT_MANAGER`
          ```
            - 같은 부서 직원의
            - 부서, 직급, 내선번호, 제한된 시스템 권한
          ```
        - 권한 : `Emp.SystemRoleCode` = `ADMIN`
          ```
          - 모든 직원의 계정/인사/상태 정보, 프로필사진, 서명파일 활성화/삭제(추가 x)
          ```
        - 권한 : `Emp.SystemRoleCode` = `EMPLOYEE`
          ```
          - 본인의
          - 내선번호, 비밀번호, 프로필사진, 전자서명파일 
          ```

### **EmpStatus**
_Enum_
**상수**
- `PENDING` : 승인보류(인사과 가입 승인 전)
- `ACTIVE` : 재직중
- `RESIGNED` : 퇴직
- `SUSPENDED` : 장기휴직 등 퇴직하지 않았지만 재직이 불가능한 상태

### **SystemRoleCode**
_Enum - ElementCollection_                 
**상수**
- `EMPLOYEE`: 일반권한
- `FRANCHISE`: 프랜차이즈 관리 권한
- `IT`: 시스템 부서 권한
- `HR`: 인사과 권한
- `DEPT_MANAGER`: 부서 관리자 권한
- `ADMIN` : 시스템 총괄 권한

**행위**
- `static register()`: 사원 등록 : EmpRegisterRequest(emp_no, empName, emp_id, rawPassword)
- `activate()` : 사원 활성화 : `Emp.status` - `ACTIVE`(활성화) + `Emp.SystemRoleCode` - `EMPLOYEE`로 설정

**규칙**
- 사원 신규 생성 시, status는 `PENDING`(보류)로 설정한다.
- PENDING 상태에서만 ACTIVATE 할 수 있다.
- 등록 완료 상태에서만 회원 정보를 수정할 수 있다.
- 개인정보를 수정할 시 현재 비밀번호가 일치한지 검사한다.
- 아이디와 사번은 중복을 허용하지 않는다.
- 비밀번호는 해시를 만들어서 저장한다.   
- 사내 이메일 주소는 『아이디 @ 도메인』 형식이다.
- 퇴직한 사원의 퇴직일을 입력하면 해당 직원의 소속정보의 end_at은 퇴직일로 입력된다.

## _emp_file_
- 사원 프로필 사진, 서명파일 등 사원 관련 파일 정보를 저장하는 엔티티

**속성**

| 필드명              | 비고             | 설명                                              |
|------------------|----------------|-------------------------------------------------|
| `file_type`      | Enum    | 파일 종류                                           |
| `original_name ` | VARCHAR(200)   | 원본명                                             |
| `stored_name   ` | VARCHAR(200)   | 서버 저장파일명(UUID)                                  |
| `extension     ` | VARCHAR(20)    | 확장자(jpg, png 등)                                 |
| `mime_type     ` | VARCHAR(100)   | image/jpeg, image/png 등                         |
| `file_size     ` | BIGINT         | 바이트 단위, 5MB(5 * 1024 * 1024 = 5,242,880B) 미만 허용 |
| `is_active     ` | bool           | 활성화여부                                           |

### **FileType**
_Enum_                                               
**상수**
- `SIGNATURE` : 전자서명
- `PROFILE_PICTURE` : 사원프로필사진

**행위**
- `static addFile()`: 파일(프로필사진, 서명 이미지) 등록 - EmpFileRequest
- `void deactivateTheOtherFiles()`: 
- `void deactivateFile()`: 파일 숨기기

**규칙**
- 파일을 추가하면 추가한 파일은 `ACTIVE`상태가 되며, 같은 타입의 파일을 비활성화한다.
- 확장자(`extension`)은 "jpg", "jpeg", "png", "gif"만 허용한다
- MIME 타입(`mime_type`)은 "image/jpg", "image/jpeg", "image/png", "image/gif"만 허용한다.
- 파일 사이즈는 5MB를 초과할 수 없다.

## _emp_belongings_
- 사원의 부서 소속 및 직급/직책 이력 정보를 저장하는 엔티티

**속성**

| 필드명                | 비고            | 설명                                    |
|--------------------|---------------|---------------------------------------|
| `dept_id`          | INT           |                                       |
| `position`         | VARCHAR(20)   | 직급                                    |
| `is_primary`       | bool          | 주요부서 (ex- 인사과면서 TF팀일시, 인사과(T), TF팀(F) |
| `start_at`         | DATE          | 소속(발령)일자                              |
| `end_at`           | DATE          | 직무해제일자                                |

**행위**
- `static registerEmpBelonging()` : dept, position, isPrimary, startAt : 새로운 소속정보를 기록
- `markEnd` : 사원의 소속정보를 해제

**규칙**
- 사원의 소속 부서/직급 변경시 이력을 새로 등록해야한다.(잘못입력시에는 물리적으로 삭제 필요)
- 등록한 이력에 대해 직무해제 일자는 향후에 등록/변경할 수 있다.
