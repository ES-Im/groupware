## _dept_
- 부서 코드와 부서 활성 여부 등 부서 기본 정보를 저장하는 엔티티

**속성**

| 필드명             | 비고           | 설명             | 
|-----------------|--------------|----------------|
| `id         `   | INT          |                |    
| `dept_code  `   | varchar(3)   | 부서코드 ex- '001' |    
| `dept_name  `   | varchar(100) | 부서명            |    
| `is_active  `   | bool         | 활성화 여부         |    
| `created_at `   | DATETIME     |                |    
| `updated_at `   | DATETIME     |                |    

**행위**
- `static registerDept()`: 부서 생성: deptCode, deptName
- `activate()`: 부서 활성화
- `deactivate()`: 부서 비활성화
- `renameDept()`: 부서 이름 변경

**규칙**
- 부서 비활성화는 해당 부서에 'ACTIVE' 상태인 사원이 없을 때만 가능하다.
- 신규 부서는 기본적으로 활성 상태이다.
- 비활성화 부서는 일반 조회 대상에서 제외된다.
- 부서코드는 부서 생성 후 변경하지 않는다.
- 부서 정보 수정 / 등록은 SystemRoleCode = `ADMIN`만 할 수 있다.

## _emp_
- 사원 계정 및 인적사항, 입사일 등 사원 기본 정보를 저장하는 엔티티
- Aggregate root

**속성**

| 필드명                | 비고               | 설명                               |
|--------------------|------------------|----------------------------------|
| `emp_no`           | varchar(11)      | 사번, 202603xxx 형식(입사연월+연번)        | 
| `emp_name`         | VARCHAR(20)      | 직원 이름                            | 
| `emp_id`           | VARCHAR(50)      | 아이디                              | 
| `emp_pw`           | varchar(50)      | 비밀번호                             | 
| `company_email`    | VO / varchar(50) | 직원의 사내 이메일(emp_id + @haruon.com) | 
| `extension_no`     | VARCHAR(20)      | 사무실 직통번호                         | 
| `status`           | Enum             | 재직상태                             | 
| `system_role_code` | SET<ENUM>        | 시스템 권한                           | 
| `hire_at`          | DATE             | 입사일                              | 
| `resign_at`        | DATE             | 퇴직일                              | 
| `emp_Files`        | List             | 퇴직일                              | 
| `emp_Belongings`   | List             | 퇴직일                              | 


#### **EmpStatus**
_Enum_
**상수**
- `PENDING` : 승인보류(인사과 가입 승인 전)
- `ACTIVE` : 재직중
- `RESIGNED` : 퇴직
- `SUSPENDED` : 장기휴직 등 퇴직하지 않았지만 재직이 불가능한 상태

#### **SystemRoleCode**
_Enum - ElementCollection_                 
**상수**
- `EMPLOYEE`: 일반권한
- `FRANCHISE`: 프랜차이즈 관리 권한
- `IT`: 시스템 부서 권한
- `HR`: 인사과 권한
- `DEPT_MANAGER`: 부서 관리자 권한
- `ADMIN` : 시스템 총괄 권한

**행위**
- `static register()`: 사원 등록 : EmpRegisterRequest(emp_no, empName, emp_id, rawPassword)
- `approveRegister()`: 가입승인
- `removeFile()`: 파일 삭제
- `changeResignedEmpInfoByAdmin()`: 퇴직자정보 수정
- `changeInfoBySelf()`: 본인 정보 수정
- `changeInfoByDeptManager()`: 부서관리자용 사원 정보 수정
- `changeInfoByAdmin()`: 총괄 관리자용 사원 정보 수정

**DTO**
- EmpAdminUpdateParam : 시스템 총괄관리자에 의한 사원 정보 변경 request
- EmpDeptManagerUpdateParam : 부서 시스템 관리자에 의한 사원 정보 변경 request
- EmpSelfUpdateParam : 사원의 개인 정보 변경 request
- EmpBelongingsParam : 사원의 소속정보 등록/수정 request
- EmpFileParam : 사원 관련 파일(서명, 프로필사진) 등록 request
- EmpFileStatusChangeParam : 사원 관련 파일(서명, 프로필 사진) 활성화/비활성화 request
- EmpRegisterParam : 사원 가입 request

**규칙**
- 비밀번호는 [영문+특수문자+숫자]포함 8자 이상이여야 하며, 해시를 만들어서 저장한다.   
- 사내 이메일 주소는 『아이디 @ 도메인』 형식이다.
- 내선번호는 000-0000 형식이다
- 아이디와 사번은 중복을 허용하지 않는다.
- 사원 신규 생성 시, status는 `PENDING`(보류)로 설정한다.
- `PENDING` 상태에서만 가입승인을 할 수 있다.
- 등록 완료 상태에서만 회원 정보를 수정할 수 있다.
- 퇴직한 사원의 퇴직일을 입력하면 해당 직원의 소속정보의 end_at은 퇴직일로 입력된다.
- 본인이 개인정보를 수정할 시 현재 비밀번호가 일치한지 검사한다.
- 시스템 권한 - 부서관리자는 부서 내 직원에게 시스템 권한(본인보다 낮은 등급만)만 부여할 수 있으며, 내선번호 수정이 가능하다
- 시스템 권한 - 총괄관리자는 모든 부서의 직원의 정보를 대부분 수정할 수 있다.

## _emp_file_
- 사원 프로필 사진, 서명파일 등 사원 관련 파일 정보를 저장하는 엔티티

**속성**

| 필드명                | 비고              | 설명                                              |
|--------------------|-----------------|-------------------------------------------------|
| `original_name `   | VARCHAR(200)    | 원본명                                             |
| `stored_name   `   | VARCHAR(200)    | 서버 저장파일명(UUID)                                  |
| `mime_type     `   | VARCHAR(100)    | image/jpeg, image/png 등                         |
| `extension     `   | VARCHAR(20)     | 확장자(jpg, png 등)                                 |
| `file_size     `   | BIGINT          | 바이트 단위, 5MB(5 * 1024 * 1024 = 5,242,880B) 미만 허용 |
| `file_type`        | Enum            | 파일 종류                                           |
| `is_active     `   | bool            | 활성화여부                                           |

### **FileType**
_Enum_                                               
**상수**
- `SIGNATURE` : 전자서명
- `PROFILE_PICTURE` : 사원프로필사진

**행위**
- `static addFile()`: 파일(프로필사진, 서명 이미지) 등록 - EmpFileRequest

**규칙**
- 파일을 추가하면 추가한 파일은 `ACTIVE`상태가 되며, 같은 타입의 파일을 비활성화한다.
- 확장자(`extension`)은 "jpg", "jpeg", "png", "gif"만 허용한다
- MIME 타입(`mime_type`)은 "image/jpg", "image/jpeg", "image/png", "image/gif"만 허용한다.
- 파일 사이즈는 5MB를 초과할 수 없다.

   ```

     - 50%이상 근로 시, 지각이나 결근으로 처리한다.
   - 출장 일정(`schedule`)이 있는 경우 근로시간을 조정하여 근태상태를 판별한다.

   - 직원이 출근/퇴근을 기록하지 않았다면 다음날 근태 마감 시 상황에 따라 근태를 기록한다
     - 일정이 있는 경우 
       - 종일 연차 : 연차
       - 종일 출근 : 정상 근무
     - 일정이 없는 경우 : 결근
   - 각 직원별 근태 엔티티는 하나만 등록하는게 원칙이며, 반차가 있을 경우 추가 근태 엔티티가 생성된다.(최대 2개)
   ```
2. 근태 수정
    ```

     - 확정은 익월 5일 전까지 완료해야한다.
     - 수정
       : 확정 이후 수정을 할 수 없다.
       : 출퇴근시간 / 일일 근태타입 수정이 가능하다. 
       : 만약 근태타입을 미지정하거나, '정상근무/결근/조퇴및지각'으로 수정한다면, 근무시간을 재 계산하여 
       : 수정시에는 수정사유를 입력해야한다.
    ```

## _Emp_Annual_Leave_
- 사원별 연차/휴가 보유량, 사용량, 잔여량을 관리하는 엔티티

**속성**


| 필드명                       | 비고               | 설명       | 
|---------------------------|------------------|----------|
| `id         `             | INT              |          |    
| `emp_id     `             | INT              |          |    
| `grant_year `             | INT              | 대상연도     |    
| `base_grant_days `        | DECIMAL(4,1)     | 총 연차 수   |    
| `additional_Grant_Days  ` | DECIMAL(4,1)     | 추가 부여 연차 |    
| `used_days  `             | DECIMAL(4,1)     | 사용한 연차 수 |    
| `created_at `             | DATETIME         |          |    
| `updated_at `             | DATETIME         |          |    

**행위**
- `static createEmpAnnualLeave()` : 매년 1월 1일 각 사원의 연차 계산하여 등록
- `recalculateAdditionalLeave()` : 추가 부여 연차를 수정
- `calculateRemainedLeaveDays()` : 잔여연차를 조회
- `useLeaveDays()` : 연차 소모

**규칙**
- 연차는 사원의 입사일을 기준으로 매년 1월 1일 산정한다.
- 연차 부여는 『근로기준법 제 60조』을 반영한다.
  - 계속근로기간이 1년 미만인 사원은 1개월 개근 시 1일의 연차(최대 11일수)를 부여한다.
  - 계속근로기간이 3년 이상인 사원은 매 2년마다 1일의 연차를 추가 부여한다
  - 최대 연차는 25일을 초과할 수 없다.
- 추가 부여 연차에 대해서는 별도 필드로 관리한다
- 추가 부여 연차를 가감할 수 있되, 마이너스가 되지 못한다.
- 연차 사용 수(used_days) 총 잔여 연차 수(base_grant_days + additional_grant_days)를 넘지 못한다
